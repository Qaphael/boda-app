# Typography Migration: Adopt Standard M3 Scale

## What changed from previous plan
The prior mapping preserved the app's existing font sizes and just renamed them to M3 tokens. That's wrong — the user wants the **actual canonical M3 type scale** from Google. So Type.kt gets rewritten with official M3 values, and screen files map semantically to the new tokens.

## Step 1: Mapping from current inline sizes → new M3 tokens

Since Type.kt sizes change, this is NOT a 1:1 size match. Map by semantic purpose:

| Current code | New token | Rationale |
|-------------|-----------|-----------|
| 32.sp + ExtraBold | `headlineLarge` | App's biggest text maps to headlineLarge (32sp in M3) |
| 28.sp + ExtraBold | `headlineMedium` | 28sp matches M3 headlineMedium |
| 24.sp + ExtraBold | `headlineSmall` | 24sp matches M3 headlineSmall |
| 20.sp + Bold | `headlineSmall` or `bodyLarge` | 20sp sits between headlineSmall (24) and bodyLarge (16) — use `bodyLarge` for regular text, `headlineSmall.copy(fontSize = 20.sp)` if heading |
| 18.sp + Bold | `bodyLarge` | 18sp→16sp bodyLarge (slight size reduction acceptable) |
| 16.sp + Bold | `bodyLarge` | Exact M3 match |
| 14.sp + Bold | `labelLarge` | M3 labelLarge is 14sp Medium — use `.copy(fontWeight = FontWeight.Bold)` for emphasis |
| 14.sp + Normal | `bodyMedium` | M3 bodyMedium is 14sp Normal |
| 13.sp + Bold | `labelLarge.copy(fontSize = 13.sp)` | Between labelLarge (14) and bodySmall (12) — edge case for avatar initials |
| 12.sp + Bold | `bodySmall` | M3 bodySmall is 12sp Normal — add `.copy(fontWeight = FontWeight.Bold)` |
| 12.sp + Normal | `bodySmall` | Exact M3 bodySmall match |
| 11.sp + Bold/Medium | `labelSmall.copy(fontWeight = FontWeight.Bold)` | M3 labelSmall is 11sp Medium |
| 11.sp + Normal | `labelSmall` | Exact M3 labelSmall match |
| 10.sp | `labelSmall.copy(fontSize = 10.sp)` | Slightly smaller than M3 labelSmall — use copy |
| 9.sp | `labelSmall.copy(fontSize = 9.sp)` | Below M3 minimum — use copy |

**Important rules:**
- When the token's default weight matches what the code needs, DROP fontWeight
- When weight differs, use `.copy(fontWeight = FontWeight.X)`
- M3 display roles (57/45/36sp) are unused — this mobile app doesn't need text that large

## Step 3: Edit pattern per file

```
// BEFORE:
Text("Gulu Boda Wallet", fontWeight = FontWeight.Bold, fontSize = 24.sp)

// AFTER:
Text("Gulu Boda Wallet", style = MaterialTheme.typography.headlineSmall)
// headlineSmall = 24sp Normal in M3 — fontWeight dropped since it's a page heading

// BEFORE:
Text("Speed Safe", fontSize = 12.sp, fontWeight = FontWeight.Bold)

// AFTER:  
Text("Speed Safe", style = MaterialTheme.typography.bodySmall.copy(fontWeight = FontWeight.Bold))
// bodySmall = 12sp Normal, needs bold override
```

After all edits: remove unused `sp` and `FontWeight` imports from each file.
